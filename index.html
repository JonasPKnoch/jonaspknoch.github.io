Projects

Bronco Text Generator

Bronco is a domain-specific programming language for procedural text generation. It allows users to author grammar-based text generators using a feature-rich, Turing-complete language. It is written as an extensible C# library and a standalone IDE, both open source on GitHub.

The language was made over a summer, funded through UROP and SPUR, under Professor Rogelio E. Cardona-Rivera and his Quantitative Experience Design Laboratory. It was the subject of a publication: Bronco: A Universal Authoring Language for Controllable Text Generation at ICIDs 2022.

Group Theory for Procedural Content Generation (GTPCG)

GTPCG is an ongoing research project exploring the potential connection between group theory and procedural content generation. More specifically, we propose that PCG artifacts can be described with groups and that such descriptions can be used for new methods of generation and analysis.

Thus far, I and Professor Rogelio E. Cardona-Rivera have been researching generation, my exhaustive execution of group actions. This work has resulted in a poster presentation, and a research paper submission. We are currently in the process of another paper submission, and releasing an extension for Blender, allowing for group theoretic generation of 3D geometry.

BIT4D

BIT4D (Back In The 4th Dimension), is a little 4-dimensional rendering engine built in Godot. Using the slicing method, it renders 4D geometry by calculating its 3-dimensional intersection with the view plane. Still in progress, it currently supports lighting and texturing, and will eventually include collision and simple physics. The current version is available on GitHub.

The rendering is done efficiently on the GPU. The 4D models are loaded from a custom file format that can be converted from 4D OFF.

4D Axonometric Projection

4D axonometric projection is a method of mapping 4 dimensions down to 2 orthographically; without perspective. I’ve made a collection of tools for rendering 4D geometry with this method. It uses a custom file format and can render in multiple formats. The project is made in Processing and is available on GitHub. A more detailed writeup of the method can also be found on the repository.

Colder On Mars

Colder On Mars is a silly little web applet I made to teach myself web programming. Using numerous APIs strung together, it compares the weather at the user’s current location, with the current weather on Mars as provided by the InSight Rover (May she rest in peace). It is hosted on AWS’s S3 and still works due to fallback data. Code available on GitHub.
